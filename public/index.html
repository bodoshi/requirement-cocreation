📚 方法论参考
三问法：Q1 根目的 → Q2 客群+问题 → Q3 ROI
六不要：感觉不错就做 / 看到别人做就跟 / 想做就做 / 被动接需求 / 想当然 / 不看数据
三看：看宏观（行业趋势）/ 看竞争（同类方案）/ 看用户（用户研究）
需求分类：简单（单一问题→单一方案）/ 常规（单一问题→多方案）/ 复杂（宽泛问题→需拆解）

💡 四种驱动类型变体
用户反馈驱动：反馈背后的真实问题 / 反馈用户代表性 / 修复成本vs影响
业务项目驱动：项目业务目标 / 服务客群 / 资源vs价值
老板需求驱动：期望达成指标 / 判断依据是否验证 / 替代方案
数据分析驱动：数据核心问题 / 影响范围频次 / 提升预期

🔗 跨模块依赖模式
D1 共享数据实体：共用实体变更需同步所有消费方
D2 上下游联动：输出=下游输入，变更需适配
D3 状态一致性：多系统状态需保持一致或定义优先级
D4 供应商共享：多系统依赖同一外部数据源
D5 边界模糊：功能归属不清导致重复/缺失

连接设置
本地代理 推荐
在自己电脑终端运行 node rc-server.js，使用 Claude Code 订阅，无需 API Key。
局域网代理
连接同事的代理服务器。让运行 rc-server.js 的同事告诉你 IP 地址。
API Key 直连
使用自己的 Anthropic API Key 直连，适合没有 Claude Code 的用户。
API Key 仅存储在浏览器本地，不会上传到任何服务器。
欢迎使用 /需求共创
这是一个 AI 驱动的交互式需求分析工具，帮助 PM 快速完成需求文档的结构化分析与验证。

使用前需要配置 AI 连接方式：
方式一：本地代理（推荐）
有 Claude Code？在终端运行 node rc-server.js 即可。
方式二：局域网代理
没有 Claude Code？问同事要代理地址（如 http://192.168.1.100:3456）。
方式三：API Key 直连
有 Anthropic API Key？直接填入即可使用。
